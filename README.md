## Возможности

1.  **Автономное принятие решений**: Агент анализирует текст запроса и решает, нужно ли искать товары, получать ли детальную информацию по одному объекту или сравнивать несколько позиций.
2.  **Имитация реального API**: В проект включена база данных с товарами (ноутбуки, смартфоны, наушники) и логика работы складских остатков.
3.  **Глубокая трассировка (Tracing)**: Сохранение каждого шага выполнения, включая входные параметры инструментов, время их работы и результат.
4.  **Аналитика производительности**: Сбор агрегированных данных о качестве работы агента за сессию.

## Архитектура

Проект состоит из нескольких ключевых модулей:

  * **Marketplace API Mock**: Имитация бэкенда маркетплейса с функциями поиска и сравнения.
  * **SimpleAgent Core**: Основной класс агента, управляющий циклом обработки запроса.
  * **MetricsCollector**: Модуль, отвечающий за сбор статистики и расчет бизнес-метрик (Success Rate, Latency и др.).

## Инструменты агента

Агент оперирует тремя основными инструментами:

  * `search_marketplace` — поиск товаров по категориям («laptop», «smartphone», «headphones»).
  * `get_product_details` — получение подробных характеристик товара по его ID.
  * `compare_products` — аналитическое сравнение нескольких товаров по цене и рейтингу.

## Система метрик

Для оценки качества работы реализованы следующие показатели:

  * **Общий процент успешности (Success Rate)**: доля запросов, в которых все инструменты сработали корректно.
  * **Среднее время выполнения (Average Execution Time)**: скорость работы функций-инструментов.
  * **Распределение инструментов**: статистика популярности тех или иных функций у пользователей.
  * **Сложность запросов**: автоматическая классификация задач на простые, средние и сложные на основе количества действий агента.

## Примеры запросов

Агент успешно обрабатывает запросы на естественном языке, такие как:

  * *"Найди лучшие ноутбуки на маркетплейсе"*
  * *"Сравни для меня топ-3 наушников"*
  * *"Какой самый дешевый смартфон доступен?"*
